Class validator.Form

This is the main form validator class. This widget will allow developers to easily transform a standard form into a fully interactive form that handle almost all invalid inputs gracefully and improve user experience.

Constructor

validator.Form ( config )
Parameters:
config <Object> Configuration object containing everything for configuring the form validator object.

Properties

List of buttons that will only enable if all the inputs on the form validator are correct.

This is used for custom inputs so their validator object can be stored in the form until the instance of is pulled out of an inline attribute

inputFields - BaseInputField[]

This is a list of input field objects that represent inputs that are to be validated using the validator.

Configuration Attributes

buttonJSON - Object[]

This will hold the JSON configuration for all buttons passed to the form validator. This will be intereated through, and the actual button objects created when the form is initialized.

checkOnSubmit - boolean

Default value is false, when set to true, indicators on inputs, and the form's status will only update when the submit button is clicked.

correctIndicatorText - string

The default text that will be used to set the innerHTML of the correct indicator when it is created dynamically. A non breakable space by default, and will be passed to any input field added to the validator if they do not already have this property set in their configuration.

createCorrectIndicator - boolean

This is a setting passed onto each field added to the validator. If this is true, the input field will create an correct indicator based setting in the configuration object passed to the field, or on the default settings passed to it from this class.

createIncorrectIndicator - boolean

This is a setting passed onto each field added to the validator. If this is true, the input field will create an incorrect indicator based setting in the configuration object passed to the field, or on the default settings passed to it from this class.

defaultCorrectIndicatorCss - string

The default css used when creating the correct indicator dynamically. This css value is '' by default, and will be passed to any input field added to the validator if they do not already have this property set in their configuration.

defaultIncorrectIndicatorCss - string

The default css used when creating the incorrect indicator dynamically. This css value is '' by default, and will be passed to any input field added to the validator if they do not already have this property set in their configuration.

defaultIndicatorDomType - string

This is the name of the type of dom object used to create the indicators. The default is SPAN

excludedButtons - string[]

List of ids of submit buttons that are exempt from the form validator. For instance, you may have a submit button that deletes the data pertaining to the record open in the form. Obviously the form should not have to be filled in correctly to do this.

fieldJSON - Object[]

This will hold the JSON configuration for all inputs passed to the form validator. This will be intereated through, and the actual input field objects created when the form is initialized.

form - HTMLElement

This is the form DOM object (or any DOM object) that surrounds the inputs of the form. (NOTE: Not all inputs HAVE to be inside the form dom if your using the JSON method to declar your fields).

incorrectIndicatorText - string

The default text that will be used to set the innerHTML of the incorrect indicator when it is created dynamically. A non breakable space by default, and will be passed to any input field added to the validator if they do not already have this property set in their configuration.

onSubmit - Function

This is a function that is called after the form is checked, and is valid, but before the actual submit takes place. If this function returns true, the form will submit, if it returns false, the form submission will not proceed.

onSubmitScope - Object

This is the scope in which the onSubmit function will be executed. If this is null, the function will be executed in the global scope.
